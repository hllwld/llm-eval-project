# 评测前置校验报告

> 对应脚本：`precheck.py`

## 概述

- **目标**：正式评测前用低成本（秒级）自动侦测测试集质量问题
- **方法**：规则扫描 + 弱模型探测 + LLM 自检，三层递进
- **输出**：pass / warn / fail 三级结论，高危直接中断评测

## 校验层级

### 第一层：规则扫描（零成本）

| 检查项 | 阈值 | 说明 |
| --- | --- | --- |
| 选项熵 | 正确答案分布 >50% 某选项 → ERROR | 防止模型蒙对 |
| 干扰项有效 | 某正确选项从未出现 → WARN | 选项顺序固化 |
| 难度分布 | hard 题 <20% → WARN | 区分度不足 |
| 答案长度偏差 | 全部参考答案长度几乎一致 → WARN | 模型学到长度特征 |

### 第二层：弱模型探测

- 用已有 review 数据统计 MCQ 正确率
- 弱模型正确率 >85% → WARN（测试集对强模型失效）

### 第三层：LLM 自检（DeepSeek API）

- 采样各难度题目，让 LLM 评估是否「过简单」或「充数题」
- 网络不可用时自动跳过

## 运行方式

```bash
cd llm-eval-project
python scripts/precheck.py
```

建议放在 CI 流水线中，作为 `full_benchmark.py` 的前置步骤：
- 通过 → 自动执行评测
- 警告 → 继续但记录
- 错误 → 中断，先修测试集
