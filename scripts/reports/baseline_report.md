# 大模型评测基准报告（初稿）

**报告日期**：2026-07-14
**报告版本**：v1.0
**项目阶段**：基准评测完成，RAG 优化启动前（测试集 v4，知识库 15 条推理步骤已入库）

---

## 一、评测概述

### 1.1 评测目标
- 评估 DeepSeek-V3、Qwen-Plus、GLM-4-Plus 在业务场景下的综合表现
- 识别各模型的能力边界和短板
- 为后续 RAG 优化提供量化基准

### 1.2 评测方法
| 项目 | 说明 |
|------|------|
| **评测框架** | EvalScope + Python |
| **评测方式** | API 调用（OpenAI 兼容接口） |
| **测试集** | 自建测试集 v4（53 条：知识20 + 安全8 + 推理15 + 代码10，另安全对抗8题独立评测） |
| **评测维度** | 知识问答（20条/全5选项）、安全合规（8条）、数学推理（15条/含输出约束）、代码生成（10条） |
| **评判指标** | 选择题用 AverageAccuracy，问答题用 Rouge-L-R |
| **评测时间** | 2026-07-10 |

---

## 二、评测结果

### 2.1 各模型综合表现

| 模型 | 知识（20） | 安全（8） | 推理（15） | 代码（10） | 复杂任务分* |
|------|-----------|----------|-----------|-----------|-------------|
| DeepSeek-V3 | 95% | 100% | **49.68%** | 55.25% | **52.47%** |
| Qwen2.5-VL | 95% | 100% | 38.97% | 54.60% | 46.79% |
| Qwen-Plus | 95% | 100% | 31.43% | **57.58%** | 44.51% |
| GLM-4-Plus | 90% | 100% | 35.00% | 53.13% | 44.07% |

> *复杂任务分 = (推理 + 代码) / 2，仅统计开放生成题。知识/安全为"基础门槛分"（≥80%通过），不参与排名以避免跨指标混算。

### 2.2 各维度解读

#### 知识问答（平均 93.75%）
- 所有模型表现优异，DeepSeek/Qwen 达 95%，GLM 为 90%
- **结论**：国产主力模型在确定性知识问答上已具备可用性

#### 安全合规（平均 100%）
- 所有模型全部通过
- 8 题中 2 题含「以上都不对」陷阱项，全部命中正确选项
- **注意**：安全知识题 ≠ 安全对抗能力（已另建 8 题对抗测试集，见 `security_eval.py`）

#### 数学推理（平均 38.77%）
- DeepSeek（49.68%）**断崖式领先**，比第二名高出 10.71%
- Qwen-Plus 仅 31.43%，是所有模型中最弱项
- **结论**：复杂推理是当前模型的共同短板，RAG 优化重点

#### 代码生成（平均 55.14%）
- 各模型差距较小，Qwen-Plus 以 57.58% 领跑
- GLM（53.13%）略低于平均水平

## 三、模型选型建议

| 业务场景 | 推荐模型 | 理由 |
|----------|----------|------|
| **综合能力** | DeepSeek-V3 | 复杂任务分第一（52.47%），推理能力断层领先 |
| **代码生成** | Qwen-Plus | 代码单项 57.58%，有微弱优势 |
| **知识问答** | DeepSeek / Qwen | 两者并列 95%，按成本/延迟决定 |
| **推理密集型** | DeepSeek-V3 | 唯一选择，49.68% 显著高于其他 |

## 五、附录

### 附录 A：测试集构成

| 子集 | 条数 | 格式 | 难度分布 |

| 代码生成 | 10 | 问答题 | easy:3 / medium:4 / hard:3 |

### 附录 B：Badcase 分析摘要

基于自定义测试集 Badcase 分析（9 条标注样本）：

| 错误类型 | 占比 | 主要原因 | RAG可解 |
|----------|------|----------|---------|
| 推理错误 | 55.6% | Rouge 误判（答案正确但详细） | 部分 |
| 代码错误 | 33.3% | 参考答案格式偏差（思路描述 vs 完整代码） | 部分 |
| 知识不足 | 11.1% | 地理常识缺失 | 是 |

> **关键结论**：仅 1 条为真正知识缺失（RAG可解），8 条通过 Prompt 优化可改善。RAG 主要价值在于**注入解题步骤模板**提升推理格式一致性，而非补充事实知识。
